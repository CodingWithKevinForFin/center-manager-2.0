Styles

Styling is used to give a unique look and feel to your dashboard, while also providing a consistency across an organization's various implementations.  All elements within the dashboard can be styled with custom font families, sizes, and colors. 

 

Panel Styles

Panels inherit from common stylesets but individual styles can be overridden on a panel-by-panel basis. When editing a panel's style, each option is by default set to inherit, but can be manually overridden. The panel styles are edited by clicking on the panel's green button and then selecting Style... 

This method is preferred when a panel needs a special styling which differs from the general layout styling.

 

Stylesets

A styleset is a self contained subset of settings for all of the style-able elements within AMI. Each styleset has a unique name and bust inherited from another styleset. The exception is the DEFAULT styleset which supplies the factory default for style for each style setting within AMI.

 

Stylesets are managed using the Dashboard > Style Manager. Stylesets created using this manager are stored within the root layout's file by default, but alternatively can be exported and directly saved to a common repository making it available across dashboards.

 

Custom CSS

While AMI has hundreds of easy-to-use style options, you can achieve complete control of web styling for forms and fields using custom CSS.  The  custom CSS is accessed from Dashboard > Custom CSS. After creating custom CSS classes, the field's can then be assigned to a given CSS class.
