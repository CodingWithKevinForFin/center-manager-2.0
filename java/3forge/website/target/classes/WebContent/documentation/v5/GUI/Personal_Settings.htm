Personal Settings

Preferences allow end users to customize a dashboard for their own personal experience. Each panel within a dashboard can have its own specific configuration, and even how the panels are arranged may be user specific. In order to support this, each panel has its own default configuration and can be overridden by the user's preferences. Additionally, custom preferences can be used to drive specific AmiScript.


Preferences of each panel can be represented by JSON. This makes it easy to manage and export/import preferences across panels and layouts.

 

Defaults

Each panel can have defaults set  when in layout editor mode. Once a default is set, that panel will "remember" how to be configured.  If a default has not been set, then the current configuration is used. In other words, if you don't set defaults then it is  important to have everything positioned just right when you save a dashboard. Alternatively, if you set default values, then it doesn't matter what the configuration of the panel looks like when you save, because it's those default values that will be used for saving. This reduces the chance of saving an accidental change to something in a dashboard. 

 

User Preferences

When an end user modifies the dashboard, they can then save these modifications to their personal preferences. And next time when they login, these preferences will be applied. End users can also export/import these preferences with other users.

 

User Preferences Namespace

Users can have access to many dashboards, each with their own user preferences. In order to properly match the user's preferences with a given dashboard there needs to be a common key for reference, which is called the user preferences NameSpace. Assigning dashboards to a namespace is done using Menu > Dashboard > Dashboard Settings
