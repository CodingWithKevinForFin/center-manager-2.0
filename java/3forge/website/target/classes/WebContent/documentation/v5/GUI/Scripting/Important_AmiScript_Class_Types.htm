Important AmiScript Class Types

Below is a sampling of the class types available within amiscript. Please visit the documentation for a full, up-to-date catalog of functions and methods under Help > AmiScript Doc > AmiScript Class Types.

Session & Layout

There is a single session object for each user that is currently logged in. Through all of dashboards, the session variable is a special predefined variable. The session stores things such as the user's name, IP address they are logged in from, etc. It also provides functionality for loading a new layout, starting a download, changing the browser title, etc.

 

A layout represents a particular layout file. Layout files are what contain the various objects of a dashboard such as panel, datamodels, as well as child layouts. Please visit Managing Dashboard Files for an understanding of how a dashboard can be made of several files.

Panels, FormFields, Menu Items.

Each panel is represented by a Panel object and there are several subtypes of panels for each type of panel available. For example, TablePanel, FormPanel, ChartPanel, TabsPanel, etc. Depending on the type of panel, certain functionality is available for manipulating the panel runtime.

 

Form Panels can contain a variety of field types, and each type is represented by a subtype of the Field object, such as the FormCheckboxField, FormTextField, etc.

StyleSet

For each custom style created inside the style manager, they can be accessed using a StyleSet object. Additionally, each panel also has an associated StyleSet for manipulating the style settings of a particular style.

Data structures (Collection,Iterator, Set,Map,List)

There are 3 collection types used for organized data. Set - a unique collection of data. List - an ordered collection of data. Map - similar to a set but each entry is also associated with a value.

Table & Row

An object representation of a SQL table. With functionality for adding/editing/updating columns, rows, and cell values. A table is made up of a list of row objects.

Error

When AMI Detects an error it will throw an Error object. This will contain details about why the error occurred and potentially an inner, causing Error.
