The Center is where real-time data processing, connection management and data virtualization happens.

  • In-memory database to create tables, stored procedures, triggers, timers, derived views, and more.
  • Java applications can connect to AMI using the AMI JDBC driver.
  • Data virtualization layer that can access datasources, including other dashboards, JDBC, and AMI's own database.

Relays can be connected to any number or combination of files, databases, secure shells (SSH), scripts and custom applications.

  • Data from custom applications and messaging buses can be streamed directly into the Relay in real-time.
  • The Center can invoke commands in the relay, which will execute them on the local operating system.
  • Utilize routing rules for dynamic relay routing to have more control & security with your data

The Web is responsible for interacting with web browsers and providing the end user experience.

  • Connect to 3rd Party BI's, custom apps, and subscribe to different tables
  • Session management through a username and password which is then forwarded to a SSO/entitlements server for authentication.
  • Unique user sessions independent to others, to control what each user is allowed to see and do.

Distribute load across Web Nodes with policies to distribute users

  • Auto-reroute users as servers go on/offline
  • Dynamically add webservers to the cluster
  • Centrally maintain secure https certificates

With the Web Manager, it's possible to uniformly manage users

  • Manage user preferences centrally
  • Auto-deploy dashboards to the entire web cluster
  • Centralized logging of user activity
