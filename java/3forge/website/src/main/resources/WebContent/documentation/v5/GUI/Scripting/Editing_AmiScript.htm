Editing AmiScript

There are 3 user interfaces in which AmiScript is edited, depending on the exact use. 

Simple Return Value

There are hundreds of places within AMI where you can supply AmiScript for determining a simple value. This can be just referencing a single variable, or perhaps calling a custom AmiScript method. For example, when editing a column, you can supply the function that will return what the user sees in the column. Use the plus arrow to the right to access available variables and functions.

AmiScript Block

This is used for editing larger blocks of code, such as callbacks and datamodels. Using ctrl+space will autocomplete partial lines. Pressing period (.) will present a list of available functions. Additionally, there is a variable tree to the right where you can conveniently reference objects within the dashboard. 

String Template Editor

Used for editing the filters and relationships generated string. In this case, you can think of the input as already being inside a string, meaning that you do not need to wrap the whole thing in quotes and the ${...} string templating syntax is available by default.
