Source Control Integration

Integrating AMI with an existing source control management product such as GIT or Perforce allows for more controlled management of dashboard files, similar to controlling source code files in a software project.

Configuration

You must first connect AMI to an existing source control system.  This is done on a per user basis. Use the source control configuration tool under Menu >  Account > Source Control Settings. End the appropriate source control type and fill in your user credentials. The base path tells AMI where it expects files to reside which are managed under source control, including sub-directories (files outside that directory will not have source control functionality).

Browsing Source Control

Files located under the configured base path will have additional options within the absolute-file browser's right click context menu and the Included Layouts tool's context menu. Generally, this includes adding, submitting and retrieving files from source control, noting the exact list of options is dependent on the type of source control system connected to. 

Diffing across history

Throughout the various tools and editors within AMI, there is the ability to view the configuration underlying the dashboard's objects. When using Source Control Integration, there is now added functionality to compare current configuration with that from prior releases. Mechanically, AMI will open the selected version(s) for comparison and retrieve the object's configuration for the file using the Unique ID (see Accessing Dashboard Objects for details).
