AMI Styling Guide

In this example, we will guide you through the various styling options that are available in AMI.

 

Assume that we have a plot of a sine function as below:

 

 

This is a simple Line plot and we can explore the visualization setup by either pressing [Center] Green Button > Plot > Edit Plot or [Top Right] Green Button > Edit Plot. By doing any of the previous steps, we open the Edit Rendering Layer window

 

 

Here we can specify the variables which are being plotted on the X axis and the Y axis. In our example, degrees and sine. There are also some default values for the Marker Size and Line Size of the plot in the number of pixels (px). We can set either of these features to the desired values. For example, setting Marker Size to 10 px.

 

 

Although the Line tab offers these basic styling features, we can do much more through the Advanced tab.

 

First, we can adjust the color, shape, and size of the Markers by using the Shape, Color, Width (px) fields under the Markers section.

 

 

We can also remove the line which connects the data points and keep only the markers by leaving the Line Size (px) from the LINES section empty.

 

 

We may as well keep only the line and also adjust its color, width, and dashes. E.g., leave Shape (under MARKERS), set Line Color to #ff007f, Line Size (px) to 2 and Line Dash Size to 2 (under LINES)

 

 

By Clicking the Green Button of the chart and then navigating through Style, we can change the background color of the chart

 

 

By selecting square markers and getting advantage of the MARKER POSITION OVERRIDE option (in this case setting Bottom to 0) we get a bar chart style

 

 

NOTE: A bar chart styling option as the above is not the same as a Bar Chart charting option. The first one is a visualization option which is created upon markers' special style and the data are numeric. The second groups data into categories, which can be either numeric or nonnumeric. It also creates a special axis for those categories. A Line or Scatter chart should not be blended with a Bar Chart

 

Instead of a fixed color for our line we can select Gradient and provide a quantity (in this case sine) which will determine the range of values based on which the gradient is changing colors we get the following.

 

 

The default Gradient option uses colors from green to red, but there is the option to specify Custom Gradient

 

 

 

 

Alternatively, we could select Formula and then apply a gradient function, e.g.
gradient(sine, -1, "#ff7f00", 1, "#007fff")


Moreover, by selecting a second line with the use of X2, Y2 under AREA options and corresponding styling, we can have an Area Chart.
